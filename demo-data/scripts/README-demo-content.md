# Вспомогательные скрипты для демонстрационного содержимого

В этой папке лежит `create-demo-content.ps1` — PowerShell-скрипт для создания демонстрационного учебного содержимого через API серверной части.

Дополнительно используются:
- `extract-npz-report-content.py` — извлекает структурированный JSON-набор из `.docx`-отчета;
- `import-npz-report-content.ps1` — импортирует этот набор в запущенную серверную часть и передает полный текст материалов в `CreateMaterialRequest.body`.

## Что делает скрипт

- создает `3` курса;
- создает `8` тем;
- создает `27` материалов;
- создает `85` вопросов и варианты ответов;
- создает `13` тестов;
- явно связывает вопросы с тестами;
- назначает активные итоговые тесты для обязательных тем;
- публикует содержимое через операции публикации.

## Предварительные условия

- запускать только на демонстрационной базе данных;
- серверная часть должна быть запущена в профиле `dev`;
- нужен пользователь с ролью `EXPERT` или `ADMIN`;
- скрипт рассчитан на чистое учебное наполнение;
- если таблицы `course/topic/material/question/test` уже заполнены, повторный запуск без очистки или осознанного разрешения нежелателен;
- исходные markdown-файлы в `demo-data/scenarios` должны оставаться согласованными между собой.

## Настройка авторизации

- открой `create-demo-content.ps1`;
- для локального демонстрационного запуска используй `$AuthMode = "DemoHeader"` и `$DemoActorId = "1"`;
- токен нужен только при запуске вне демонстрационного dev-контура с `$AuthMode = "Bearer"`;
- если используется режим bearer, замени `$Token = "<PASTE_TOKEN_HERE>"` на действительный токен;
- значение `$BaseUrl` должно соответствовать адресу запущенной серверной части.

## Рекомендуемая предварительная проверка

- сначала проверь текущего пользователя запросом `GET /api/v1/me`;
- если этот вызов не работает, не запускай создание содержимого;
- если в базе уже остались частично записанные данные после ошибки, лучше начать с новой демонстрационной базы.

## Замечания по безопасности

- скрипт предназначен только для демонстрационной базы;
- при первой ошибке выполнение останавливается, чтобы не повредить локальную карту идентификаторов;
- диагностические файлы запросов и ответов сохраняются в `tmp/demo-content-run-logs`;
- если вызов завершился ошибкой, сначала изучи соответствующие `*.request.json`, `*.response.json` и `*.error.txt`, а уже потом меняй передаваемые данные;
- `create-demo-content.ps1` использует короткие описательные данные материалов из `demo-data/scenarios/materials-source.md`;
- `import-npz-report-content.ps1` предназначен для загрузки содержимого из отчета и отправляет полный учебный текст в `material.body`;
- если серверная часть отклоняет `scoringPolicyCode = "DEFAULT"`, останови запуск и сначала перепроверь текущее состояние системы.

## После выполнения

- после неудачного частичного запуска рекомендуется заново подготовить демонстрационную базу, затем выполнить инициализацию организации, пользователей и только потом загрузку содержимого;
- сверь результат с `demo-data/scenarios/acceptance-checkpoints.md`;
- перед переходом к назначению обучения и пользовательским сценариям проверь количество записей, состояние публикации, связи с итоговыми тестами и изоляцию пользовательских данных.
